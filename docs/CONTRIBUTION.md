# Contribution and authorship

## Luca Panayiotou

Luca Panayiotou identified false completion as a practical agent failure mode and set the core acceptance principle: a consequential task is not complete merely because an agent or tool reports success; completion must correspond to independently checked state.

For Agent Reliability Arena, Luca directed the comparison toward an employer-readable question: whether different functional forms of intelligence inside one unified system—planning, execution, audit, recovery and synthesis—improve reliability enough to justify their cost.

## AI-assisted implementation

The repository's architecture, Python implementation, deterministic tests, documentation and static viewer were developed through AI-assisted engineering under Luca's direction. AI assistance does not replace evidence: behaviour is specified in tests, published artifacts are reproducible, and release claims are limited to what the verifier and fixtures establish.

## What the project does not claim

The project does not claim that deterministic role policies are external-model results, that the specialist condition is universally superior, or that role-conditioned components are independent conscious beings.

Future real-model work must preserve exact versions, prompts, usage, repeated paired runs and uncertainty before comparative conclusions are published.
