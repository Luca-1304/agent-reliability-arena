04 / PUBLIC PROJECTS

Two projects. One reliability thesis.

An AI agent should not be trusted because it sounds certain. Completion and orchestration should be evaluated against observable evidence.

PUBLIC · PYTHON · MODEL-AGNOSTIC01

Agent Completion Verifier

A small evaluation harness for detecting false completion—when an agent says a task is finished without enough evidence that the required external action succeeded.

  • Four explicit outcomes: verified, partial, unverified and failed
  • Ordered retry, recovery, regression and rollback handling
  • Generic and OpenAI-style trace adapters
  • Controlled failure-injection benchmark
  • Confined sandbox with independently observed file postconditions
