    07 / A REALISTIC FIRST 90 DAYS

    How the capability becomes useful inside a team.

    This is an operating approach, not a promise made without seeing the organisation’s systems, constraints and risk profile.

    DAYS 1–30

    Map the real failure surface.

    Understand the workflow before recommending AI.

    • Interview process owners
    • Collect representative failures
    • Define acceptance contracts
    • Establish baseline quality and review load
    DAYS 31–60

    Build a narrow shadow pilot.

    Test one valuable workflow without granting unsafe authority.

    • Create evaluation cases
    • Prototype the narrowest useful AI role
    • Keep consequential decisions human-approved
    • Measure errors and recovery
    DAYS 61–90

    Turn results into a decision.

    Show where to scale, redesign or stop.

    • Publish reproducible findings
    • Separate provider and workflow failures
    • Prioritise next use cases
    • Leave a roadmap and ownership model
